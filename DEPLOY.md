# คู่มือติดตั้งใช้งานจริงที่ร้าน (Go-Live Guide)

ใช้เวลาประมาณ 30–45 นาที ไม่ต้องมีเซิร์ฟเวอร์เอง ค่าใช้จ่ายเริ่มต้น **฿0** (ใช้ฟรีทั้ง Supabase และ Vercel ได้จนกว่าคนจะเยอะมาก)

ภาพรวม 4 ขั้นตอน:
```
1. ฐานข้อมูล (Supabase)  →  2. ขึ้นเว็บ (Vercel)  →  3. ตั้งค่าร้าน  →  4. พิมพ์ QR ติดโต๊ะ
```

---

## ขั้นที่ 1 — ตั้งฐานข้อมูล Supabase

1. สมัคร/ล็อกอินที่ [supabase.com](https://supabase.com) → **New project**
   - ตั้งชื่อ, ตั้ง Database password (เก็บไว้), เลือก region ใกล้ไทยที่สุด (เช่น Singapore)
2. รอสร้างเสร็จ (~2 นาที) → ไปที่เมนู **SQL Editor** → **New query**
3. เปิดไฟล์ `supabase/schema.sql` ในโปรเจกต์นี้ → **คัดลอกทั้งหมดไปวาง → กด Run**
   - คำสั่งนี้สร้างตาราง, ความปลอดภัย (RLS), ระบบจับคู่, ระบบแบน, เปิด realtime และตั้ง cron ลบข้อมูลให้ครบในครั้งเดียว
4. เปิด **Anonymous sign-in**: เมนู **Authentication → Sign In / Providers → Anonymous** → เปิดสวิตช์
5. เก็บค่า 2 อย่างจาก **Project Settings → API**:
   - `Project URL`
   - `anon` / `publishable` key
   > ใช้แค่ anon key เท่านั้น **ห้ามเอา service_role key ไปใส่ฝั่งเว็บ** เด็ดขาด

---

## ขั้นที่ 2 — ขึ้นเว็บด้วย Vercel

วิธีที่ง่ายสุดคือผ่าน GitHub:

1. ดันโค้ดขึ้น GitHub (สร้าง repo ใหม่แล้ว push โฟลเดอร์นี้)
2. ไปที่ [vercel.com](https://vercel.com) → **Add New → Project** → เลือก repo
3. ที่หน้า config ใส่ **Environment Variables** 3 ตัว:

   | Name | ค่า |
   |---|---|
   | `NEXT_PUBLIC_SUPABASE_URL` | Project URL จากขั้นที่ 1 |
   | `NEXT_PUBLIC_SUPABASE_ANON_KEY` | anon key จากขั้นที่ 1 |
   | `QR_SECRET` | สตริงสุ่มยาวๆ (สร้างเองอะไรก็ได้ เก็บเป็นความลับ) |

4. กด **Deploy** → รอสักครู่จะได้ลิงก์ เช่น `https://your-bar.vercel.app`
5. (แนะนำ) ผูกโดเมนของร้านเอง เช่น `chat.barname.com` ที่เมนู **Settings → Domains**

> **ไม่ใช้ GitHub ก็ได้:** ติดตั้ง Vercel CLI แล้วสั่ง `npx vercel` ในโฟลเดอร์โปรเจกต์ ตอบคำถามแล้วตั้ง env vars ทีหลังที่ dashboard

## ขั้นที่ 3 — ตั้งค่าให้ตรงกับร้าน

แก้ 3 จุดในโค้ดให้ตรงกับร้านจริง แล้ว push ใหม่ (Vercel จะ deploy ให้อัตโนมัติ):

**1. พิกัด + รัศมี geofence** — ไฟล์ `app/actions.ts` ตัวแปร `BARS`
```ts
const BARS = {
  demo: { lat: 13.7563, lng: 100.5018, radiusMeters: 120 },
  // เปลี่ยน lat/lng เป็นพิกัดร้าน (ดูจาก Google Maps คลิกขวาที่จุดร้าน)
  // radiusMeters = รัศมีที่ยอมให้เข้า (เช่น 80–150 ม. ครอบคลุมทั้งร้าน)
};
```
> ถ้ายังไม่อยากใช้ GPS ตอนนี้ ปล่อยไว้ก่อนได้ — ระบบจะข้ามการเช็คถ้าผู้ใช้ไม่อนุญาต location ตัวกัน QR รายวันยังทำงานอยู่

**2. เวลา cron ลบข้อมูล** — ไฟล์ `supabase/schema.sql` ตั้งไว้ตี 4 เวลาไทยแล้ว (`0 21 * * *` = 21:00 UTC) ถ้าร้านปิดเวลาอื่นปรับเลขชั่วโมง (เวลาไทย − 7 = เวลา UTC)

**3. รหัสบาร์ (`bar_id`)** — ตอนนี้ใช้ `demo` ทั้งระบบ ถ้ามีร้านเดียวใช้ค่านี้ได้เลย ถ้าหลายสาขาให้ตั้งรหัสต่างกันต่อสาขา (เช่น `thonglor`, `ekkamai`)

## ขั้นที่ 4 — สร้าง & พิมพ์ QR ติดโต๊ะ

รันบนเครื่องคุณ (ใส่โดเมนจริงที่ได้จาก Vercel):
```bash
npm install
npm run qr -- --base https://your-bar.vercel.app --bar demo --tables 12
```
จะได้โฟลเดอร์ `qr-codes/`:
- รูป QR แต่ละโต๊ะ (`demo-table-01.png` ...)
- `index.html` — เปิดในเบราว์เซอร์แล้วสั่งพิมพ์ ได้การ์ดตั้งโต๊ะสวยๆ พร้อมใช้

QR เหล่านี้ชี้ไปที่ `/t/demo` ซึ่งเซิร์ฟเวอร์จะเติม token ของวันนั้นให้เอง → **พิมพ์ครั้งเดียวใช้ได้ตลอด ไม่ต้องเปลี่ยนทุกวัน**

## เช็กก่อนเปิดใช้จริง (Checklist)

- [ ] รัน `schema.sql` สำเร็จ ไม่มี error
- [ ] เปิด Anonymous sign-in แล้ว
- [ ] env vars ครบ 3 ตัวบน Vercel
- [ ] ลองสแกน QR ด้วยมือถือ 2 เครื่อง → เห็นกันใน Lobby
- [ ] เครื่องหนึ่งส่ง Cheers, อีกเครื่องส่งกลับ → ขึ้น Match
- [ ] ลองส่ง QR ให้คนนอกร้าน (อยู่ไกล) → เข้าไม่ได้ (ถ้าเปิด geofence)
- [ ] เช็กว่า cron ลบข้อมูลทำงาน (ดูได้ที่ Supabase → Database → Cron Jobs)

## ค่าใช้จ่าย & การดูแล

- **Supabase Free:** พอสำหรับร้านเล็ก-กลาง (ผู้ใช้พร้อมกันหลักร้อย) ถ้าโตค่อยอัปเป็น Pro (~$25/เดือน)
- **Vercel Hobby:** ฟรีสำหรับใช้งานทั่วไป
- **การดูแล:** แทบไม่ต้องทำอะไร ข้อมูลลบเองทุกคืน ถ้าจะอัปเดตหน้าตา/ฟีเจอร์ ก็แค่ push โค้ดใหม่
